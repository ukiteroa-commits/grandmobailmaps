# 🎮 GMRP 32 — Дашборд сервера Grand Mobile Role Play

Полноценный веб-дашборд для **32-го сервера Grand Mobile RP (CRMP / GTA 5 RP)**:
интерактивная карта посёлков, маркет авто и акций, торговая площадка, гайды по ивентам и чаты посёлков.

![Стек](https://img.shields.io/badge/React-Vite-7c3aed) ![Стек](https://img.shields.io/badge/Express-Node.js-3b82f6) ![Стек](https://img.shields.io/badge/Tailwind-CSS-06b6d4)

---

## ✨ Возможности

| Вкладка | Описание |
|---|---|
| 🗺️ **Карта** | Реальная карта GMRP (склейка из игровых скриншотов) с пульсирующими неоновыми маркерами 16 посёлков, подписями и зумом. Клик → модалка с реальными диапазонами домов и квартир (сквозная нумерация: дома №1-388, квартиры №1-760) |
| 🚗 **Авто/Аксы** | Маркет машин и акций предприятий 32 сервера. Кнопка «Купить» копирует контакт продавца |
| 💰 **Торг** | Подача объявлений (сервер 32/38/Другой), лента «новые сверху», фильтр по серверам, удаление |
| 🎪 **Ивенты** | 9 карточек-гайдов (бомж пакет, нелегалы, ограбления, инкас и др.) с аккордеоном и YouTube-плеером |
| 💬 **Чаты** | Выбор сервера (из профиля игрока) → общий чат + чаты 16 посёлков. Реальные сообщения от своего ника, история на бэкенде, автообновление |
| 👤 **Профиль** | Регистрация/вход, мои объявления (Торг/Авто/Акции) с удалением, мои сервера |

## 🛠️ Стек технологий

- **Frontend:** React 18 (Vite), Tailwind CSS, React Router v6, Framer Motion, Lucide React
- **Backend:** Node.js + Express.js
- **БД:** JSON-файлы в `/server/data/` (структура «таблица = файл» — легко мигрируется на SQLite)
- **Дизайн:** тёмный неон (#7c3aed / #3b82f6 / #06b6d4), шрифт Inter, PWA-манифест, mobile-first (390×844)

## 🚀 Запуск

Нужен **Node.js 18+**.

### 1. Бэкенд (порт 3001)

```bash
cd server
npm install
npm start          # или npm run dev (с авто-перезапуском)
```

### 2. Фронтенд (порт 5173)

В другом терминале:

```bash
cd client
npm install
npm run dev
```

Откройте **http://localhost:5173** — Vite проксирует запросы `/api/*` на бэкенд, ничего настраивать не нужно.

### Продакшн-сборка клиента

```bash
cd client
npm run build      # результат в client/dist
```

## 🔌 API

| Метод | Эндпоинт | Описание |
|---|---|---|
| GET | `/api/cars?server=32` | Список машин (фильтр по серверу) |
| POST | `/api/cars` | Добавить машину `{ server, title, plate, price, contact, image_url }` |
| GET | `/api/stocks?server=32` | Список акций |
| POST | `/api/stocks` | Добавить акцию `{ server, business_name, stock_number, price_per_share, contact }` |
| GET | `/api/trades?server=32` | Объявления (без `server` — все) |
| POST | `/api/trades` | Подать объявление `{ server, nickname, item, price, contact }` → `id` (uuid) и `date` добавляются автоматически |
| DELETE | `/api/trades/:id` | Удалить объявление |
| GET | `/api/events` | Список гайдов по ивентам |
| POST | `/api/events` | Создать/обновить гайд (для админа) |
| POST | `/api/auth/register` | Регистрация `{ nickname, password, servers: ["32","38","other"] }` → `{ token, user }` |
| POST | `/api/auth/login` | Вход `{ nickname, password }` → `{ token, user }` |
| GET | `/api/auth/me` | Текущий пользователь (заголовок `Authorization: Bearer <token>`) |
| PUT | `/api/auth/me` | Обновить свои сервера `{ servers }` |
| POST | `/api/auth/logout` | Выход (сброс токена) |
| GET | `/api/messages?server=32&chat=general` | Сообщения чата (последние 100) |
| POST | `/api/messages` | Отправить сообщение `{ server, chat, text }` (требуется вход) |

### Авторизация

- Регистрация: игровой ник + пароль + выбор серверов, на которых играешь.
- Пароли хранятся как scrypt-хеши, сессия — bearer-токен в `localStorage`.
- Подача объявлений (торг/авто/акции) и сообщения в чатах — только для вошедших; ник подставляется из аккаунта.
- Удалять можно **только свои** объявления (проверка на бэкенде).
- Вкладка Чаты показывает только сервера из профиля игрока; сервера можно менять в профиле.
- Профиль (иконка в шапке) — свои объявления по разделам Торг/Авто/Акции, смена серверов, выход.

Пример добавления машины:

```bash
curl -X POST http://localhost:3001/api/cars \
  -H "Content-Type: application/json" \
  -d '{"server":"32","title":"Audi RS6","plate":"О666ОО 32","price":"3100000","contact":"@rs6_seller"}'
```

## 🗄️ Миграция на SQLite

Структура данных уже плоская: один JSON-файл = одна таблица, один объект = одна строка.
Для миграции достаточно:

1. Создать таблицы `cars`, `stocks`, `trades`, `events` с теми же полями.
2. Заменить `readTable`/`writeTable` в `server/utils/db.js` на запросы `better-sqlite3` — роуты менять не придётся.
